# Trading Assistant Bot + Mini App

Личный Telegram-ассистент для учета криптовалютных сделок, контроля риска, торговых сессий и анализа рынка Binance.

Проект состоит из двух частей:

- Telegram-бот фиксирует сделки, заметки и скриншоты, наблюдает stop/take по публичным данным и присылает уведомления.
- Mini App показывает живые цены, сделки, графики, журнал, торговые сессии, аналитику и мультитаймфреймовый разбор входа.

> Проект не обещает прибыль и не дает гарантированных сигналов. Rule score — эвристическая оценка по правилам, а не вероятность исхода.

## Основные возможности

- сделки Long/Short с entry, stop, take, количеством монет и плечом;
- наблюдение достижения stop/take без ложного утверждения об исполнении ордера;
- живой PnL в USDT и процентах;
- изменение открытой сделки и перенос stop/take;
- фотографии до/после сделки из Telegram и Mini App;
- черновик сделки по скриншоту TradingView/панели заявки через `/open` с уточнениями перед записью;
- сохранение свечей и повтор движения закрытой сделки;
- торговые сессии с отдельным стартовым депозитом, целью и статистикой;
- автоматические теги сессии и монеты;
- дневник с поиском по монете, дате и тексту;
- сканер ликвидных и волатильных монет;
- анализ 1D / 1H / 15M / 5M;
- расчет размера позиции от допустимого риска;
- русские названия монет: `биток`, `эфирка`, `солянка`, `солик`;
- защита от явно ошибочных уровней стопа.

## Команды Telegram

```text
/open
Монета: SOL
Сторона: лонг
Цена входа: 70.9
Стоп: 69.8
Тейк: 73
Количество позиций: 1.4
Плечо: 1
Причина входа: отбой от уровня
/edit 12 стоп 70.1 тейк 74 количество 1.2 5m перенес стоп после подтверждения
/note SOL ошибка: вошел без подтверждения
/trades
/close 12 73.8
/stats
/miniapp
```

Чтобы добавить фотографию к существующей сделке, отправь фото с подписью:

```text
/edit 12 стоп 70.1 тейк 74 комментарий
```

Чтобы открыть сделку по скрину, отправь `/open`, затем одно фото TradingView/панели заявки. Бот попробует извлечь монету, сторону, вход, стоп, тейк, количество и плечо. Если не хватает данных или причины входа, он задаст уточняющий вопрос и не будет записывать сделку молча. Ручной режим с фото по подписи `/open ...` остаётся доступен.

## Локальный запуск

```bash
cp .env.example .env
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python scripts/migrate.py data/trading_bot.sqlite3
python -m trading_bot.main
```

Mini App запускается отдельно:

```bash
source .venv/bin/activate
uvicorn trading_bot.web_app:app --host 127.0.0.1 --port 8080
```

Кабинет открывается через Telegram: Mini App передаёт подписанный `initData`. URL с `user_id` не поддерживается.

## Настройки

```env
TELEGRAM_BOT_TOKEN=token_from_botfather
ALLOWED_TELEGRAM_USER_IDS=123456789
APP_ENV=production
AUTO_MIGRATE=false
BUSINESS_TIMEZONE=Europe/Moscow
DATABASE_PATH=data/trading_bot.sqlite3
MARKET=futures
TOP_LIMIT=10
ALERT_POLL_SECONDS=30
OPENAI_API_KEY=
OPENAI_VISION_MODEL=gpt-5.5
WEB_APP_URL=http://127.0.0.1:8080
WEB_HOST=127.0.0.1
WEB_PORT=8080
```

Для запуска Mini App непосредственно внутри Telegram нужен публичный HTTPS URL.

`OPENAI_API_KEY` нужен только для авторазбора скриншотов. Если ключ не задан, `/open` продолжает работать по ручному шаблону.

## Хранение данных

SQLite хранит пользователей, сделки, торговые сессии, свечи, вложения, журнал, алерты, планы и результаты проверки сетапов. Токен Telegram и локальная база не попадают в Git.

## Obsidian

Проект предусматривает переносимый экспорт в Obsidian vault: Markdown с YAML properties, внутренними `[[links]]`, папками по сессиям/сделкам/дневнику и `.canvas`-картами связей. Экспорт доступен через Mini App в разделе аналитики и API `GET /api/export/obsidian.zip`. Технический дизайн: [docs/obsidian-export.md](docs/obsidian-export.md).

## Модели и агенты

Авторазбор скриншотов и будущие LLM-функции проектируются как управляемые подключения моделей в Mini App: облачные API, OpenAI-compatible local endpoints, выбор активной модели по задаче и безопасное хранение ключей на backend. В Mini App добавлен UI-скелет вкладки `Модели`; реальное сохранение ключей требует backend-миграции. Дизайн: [docs/model-connections.md](docs/model-connections.md).
