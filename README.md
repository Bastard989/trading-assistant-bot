# Telegram trading assistant bot

Бот для личного трейдинг-дневника, риск-менеджмента и ценовых уведомлений.

Он не дает гарантированных сигналов и не обещает прибыль. Его задача — помогать считать риск, фиксировать сделки, собирать статистику и быстро смотреть активные монеты.

## Что умеет

- `/defaults account risk%` — сохранить депозит и риск по умолчанию.
- `/price BTC` — получить текущую цену.
- `/top` — топ USDT-монет по смеси ликвидности и волатильности за 24 часа.
- `/sentiment BTC` — long/short ratio по Binance Futures.
- `/distance BTC 64000 66000` — измерить, на сколько процентов цена ушла от уровней.
- `/risk BTC long entry stop target account risk% leverage` — рассчитать позицию от риска.
- `/trade BTC long entry stop target qty leverage заметка` — проверить сделку и сохранить, если она проходит риск-фильтр.
- `/close trade_id exit_price fees заметка` — закрыть сделку и посчитать PnL.
- `/trades [open|closed|cancelled]` — список сделок.
- `/stats` — статистика прибыли/убытков, winrate и разрез по монетам.
- `/alert BTC >= 65000` — уведомление, когда цена достигнет уровня.
- `/alerts` — список активных алертов.
- `/journal BTC win описание` — запись в дневник трейдера.
- `/entries [BTC]` — последние записи дневника.
- `/context BTC 1D long levels=64000,68000 structure=uptrend` — сохранить старший контекст по монете.
- `/contexts [BTC]` — посмотреть сохраненный контекст.
- `/watch BTC ETH SOL` — список монет на наблюдение.
- `/plan BTC,ETH,SOL 3 50 текст` — план дня: монеты, лимит открытого риска в %, дневной стоп в USDT.
- `/miniapp` — открыть Telegram Mini App кабинет.
- Фото сделки можно отправить с подписью: `/journal BTC loss описание`.
- Скрин старшего таймфрейма можно отправить с подписью: `/context BTC 4H long levels=64000,68000`.

## Быстрый запуск

1. Создай бота у [@BotFather](https://t.me/BotFather) и получи токен.
2. Скопируй настройки:

```bash
cp .env.example .env
```

3. Впиши токен в `.env`.
4. Установи зависимости:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

5. Запусти:

```bash
python -m trading_bot.main
```

## Настройки

```env
TELEGRAM_BOT_TOKEN=put_your_bot_token_here
DATABASE_PATH=data/trading_bot.sqlite3
MARKET=futures
TOP_LIMIT=10
ALERT_POLL_SECONDS=30
WEB_APP_URL=http://127.0.0.1:8080
WEB_HOST=127.0.0.1
WEB_PORT=8080
```

- `MARKET=futures` использует Binance Futures public API.
- `MARKET=spot` использует Binance Spot public API, но `/sentiment` доступен только для futures.
- `ALERT_POLL_SECONDS` задает частоту проверки цен.
- `WEB_APP_URL` — адрес Mini App. Для настоящего Telegram Mini App нужен HTTPS URL.

## Mini App

Mini App читает ту же SQLite-базу и показывает:

- Dashboard: PnL, winrate, открытый риск, активные алерты.
- Сделки: таблица сделок с фильтром по статусу.
- Калькулятор: расчет позиции без команд.
- Контекст: старшие таймфреймы, bias, уровни и заметки.
- Дневник: записи и привязка к скриншотам через Telegram `file_id`.

Локальный запуск:

```bash
source .venv/bin/activate
uvicorn trading_bot.web_app:app --host 127.0.0.1 --port 8080
```

Открыть в браузере:

```text
http://127.0.0.1:8080/?user_id=твой_telegram_id
```

Для запуска внутри Telegram нужно опубликовать Mini App на HTTPS-домене или через туннель, затем прописать этот URL в `WEB_APP_URL`.

## Trade Review

Перед сохранением `/trade` бот оценивает сделку:

- старший контекст 1D/4H/1H/15M против или за сделку;
- расстояние до entry, stop, target и сохраненных уровней;
- R/R;
- риск в процентах от депозита;
- дневной лимит риска и дневной стоп;
- входит ли монета в watchlist и план дня;
- long/short настроение рынка.

Если сделка спорная или опасная, бот не сохраняет ее сразу. Он показывает:

- вероятностную оценку успеха;
- вероятностную оценку провала;
- причины, почему сделку стоит остановить;
- кнопку `Игнорировать и внести`;
- кнопку `Сохранить как идею`;
- кнопку `Отменить`.

Эти проценты — дисциплинарная оценка по твоим правилам и контексту, а не гарантия рынка.

## Примеры

Сохранить депозит 1000 USDT и риск 1%:

```text
/defaults 1000 1
```

Рассчитать позицию:

```text
/risk BTC long 65000 64000 68000 1000 1 5
```

Сохранить сделку:

```text
/trade BTC long 65000 64000 68000 0.01 5 пробой уровня
```

Закрыть сделку:

```text
/close 1 67200 1.5 вышел у сопротивления
```

Поставить алерт:

```text
/alert ETH <= 3000
```

Добавить старший контекст:

```text
/context BTC 1D long levels=64000,68000 structure=uptrend цена держит восходящую структуру
```

План дня:

```text
/watch BTC ETH SOL
/plan BTC,ETH,SOL 3 50 работаю только от подтвержденных уровней
```

## Данные

Все личные данные хранятся в SQLite:

- `users` — настройки пользователя.
- `alerts` — ценовые уведомления.
- `trades` — сделки и PnL.
- `journal_entries` — дневник, описания и Telegram `file_id` скриншотов.
- `market_contexts` — старший контекст, уровни, bias и скриншоты.
- `watchlist` — монеты на наблюдение.
- `daily_plans` — план дня и лимиты риска.
- `pending_trades` — сделки, остановленные review до твоего решения.
- `trade_reviews` — история оценок сделок.

Потом из этой базы можно сделать экспорт, аналитику по паттернам, учебник по удачным/неудачным сделкам или подключить отдельного агента для разбора истории.
